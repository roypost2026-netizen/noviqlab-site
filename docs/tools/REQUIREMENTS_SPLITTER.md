# Image Splitter 要件定義書 v1.0

**プロジェクト**: NoviqLab Tools
**ツール名**: Image Splitter（画像分割ツール）
**URL パス**: `/tools/image-splitter`
**作成日**: 2026-04-21
**ステータス**: Draft

---

## 1. 概要

縦長の画像（主にWebサイトのキャプチャ画像）を、印刷や共有に適したサイズに**賢く分割**するブラウザ完結型ツール。機械的な等分割ではなく、**余白行を検出して文字や図表を避けた切断位置**を自動で選ぶ「スマート分割」を搭載する。

---

## 2. 背景・課題

### 2.1 ユーザーが困っている問題

- **Webサイトのキャプチャ画像**（GoFullPage 等で取得）は縦に長く、そのままでは印刷時に豆粒のように縮小される
- 既存の画像分割ツールは：
  - **機械的に固定ピクセルで分割** → 文字が泣き別れになる
  - **オンラインサービス** → 機密情報を含むキャプチャをアップロードできない
  - **Photoshop / GIMP** → 毎回手作業、技術者以外には難しい
- **「画像のまま」分割したい需要**が無視されている（既存ツールの多くは PDF 変換が前提）

### 2.2 想定ユーザー

- Web制作者（クライアント提出用のキャプチャ印刷）
- UXデザイナー（デザインレビュー資料作成）
- マーケター（競合サイトの資料化）
- 社内事務員（Webページの印刷資料化）

### 2.3 他ツールとの差別化

| 観点 | 他ツール | Image Splitter |
|---|---|---|
| プライバシー | 多くはサーバー送信 | **ブラウザ完結・サーバー送信ゼロ** |
| 出力形式 | PDF が多い | **画像形式（PNG/JPEG/WebP）** |
| 分割ロジック | 機械的等分 | **余白行検出のスマート分割** |
| 使い勝手 | CLI or 複雑な UI | **ドラッグ＆ドロップでシンプル** |

## 3. 機能要件

### 3.1 入力

- **対応形式**: PNG / JPEG / WebP
- **単一画像処理**: 複数画像の一括処理は本Phaseでは対象外
- **入力方式**:
  - ドラッグ＆ドロップ
  - ファイル選択ボタン
- **ファイルサイズ制限**: 実用上ブラウザのメモリ依存（明示的な上限は設けない）

### 3.2 分割モード

#### 3.2.1 高さピクセル固定モード

- ユーザー指定の固定ピクセル高さで分割
- 入力例: `1000px` ごとに分割
- **デフォルト値**: 1000px

#### 3.2.2 A4印刷用モード

- A4用紙の縦横比（1:√2 ≈ 1:1.414）に合わせて自動計算
- 入力画像の**幅**から、適切な高さを逆算
- 例: 幅1200px → 高さ約1697px ごとに分割
- ユーザーは**何も入力する必要がない**（幅から自動計算）

### 3.3 スマート分割アルゴリズム

#### 3.3.1 動作仕様

- **デフォルト: ON**
- **ユーザーがトグルで OFF 可能**（OFF時は機械的分割）
- OFF時は指定ピクセル or A4計算値で正確に等分

#### 3.3.2 アルゴリズム（ON時）

1. **画像をNumPy相当のピクセル配列（Canvas ImageData）に変換**
2. **各行ごとに色の分散を計算**
3. **「余白行」を判定**:
   - 水平方向のピクセル値が全て同一色（または許容差以内）
   - 許容差: 内部固定値（RGB各値 ±10）
4. **理想の分割位置の前後 ±50px（探索範囲）を検索**
5. **探索範囲内で最も「余白度が高い」行を選択**
6. **探索範囲内に余白がなければ**: 機械的分割位置で切る（フォールバック）

#### 3.3.3 ユーザーに公開するパラメータ

**公開**:
- 高さピクセル（モード3.2.1時）
- スマート分割のON/OFFトグル

**非公開（内部固定値）**:
- 許容色差: ±10
- 探索範囲: ±50px

※ これらはJobsイズムに基づき、ユーザーに選ばせない（シンプルさ優先）。

### 3.4 プレビュー

- **アップロード直後**: 元画像をプレビュー表示
- **分割位置プレビュー**:
  - 計算された分割位置を、**画像上に赤線を重ねて表示**
  - スマート分割ON時: 赤線の位置が余白行に吸着
  - スマート分割OFF時: 赤線は正確に等分位置
- **プレビュー更新**: 分割モード・高さピクセル・スマート分割トグル変更時に即時再計算

### 3.5 出力

#### 3.5.1 出力形式

- **元と同じ形式で出力**（入力が PNG なら PNG、JPEG なら JPEG、WebP なら WebP）
- 形式変換が必要なユーザーは Image Converter と併用してもらう

#### 3.5.2 ファイル名規則

- 元ファイル名 + `-part01`, `-part02`, ... の連番
- 例: `capture.png` → `capture-part01.png`, `capture-part02.png`
- 桁数は分割数に応じて自動調整（100ピース以上なら `-part001` など）

#### 3.5.3 ダウンロード方式

- **個別ダウンロード**: 各ピースを個別にDL
- **ZIP 一括ダウンロード**: 全ピースを1つのZIPで
- 両方提供

## 4. 非機能要件

### 4.1 パフォーマンス

- 画像サイズの上限を設けないが、**ブラウザのメモリ制約に依存**
- 5000px × 縦長画像（例: 10000px）までは快適動作を目標
- 分割処理の目標: **2秒以内**（分割数10ピース程度の場合）
- スマート分割の余白検出: 画像を1回スキャン（O(width × height)）、NumPy相当の高速処理を Canvas API で実装

### 4.2 プライバシー

- **サーバー送信ゼロ**: 完全ブラウザ内処理
- **Cookie 不使用**
- **localStorage は最小限利用**:
  - 直近の設定値（高さピクセル、スマート分割ON/OFF）の保存のみ
  - 画像データは保存しない
- NoviqLab の Tools ページ共通プライバシー方針に準拠

### 4.3 UX設計

- **Jobs イズム徹底**:
  - 説明書なしで使える UI
  - デフォルトで賢い挙動（スマート分割 ON）
  - 専門用語を最小化
- **説明文**:
  - 中学生向け + 技術者向けの2層構造
  - UsageToggle コンポーネント利用（既存資産）
  - [使い方] クリックで展開
- **一貫性**:
  - Image Converter と同じデザイン言語
  - 同じカラー、同じフォント、同じ余白
  - Breadcrumb、InfoAccordion 等の既存コンポーネント流用

## 6. 技術スタック

### 6.1 フレームワーク

- **Next.js 16.2.2（App Router）**: 既存 NoviqLab サイトと統一
- **TypeScript strict mode**
- **Tailwind CSS**: スタイリング

### 6.2 画像処理

- **Canvas API**: 画像の描画・切り出し

### 6.3 ZIPダウンロード

- **JSZip**: 既存の Image Converter と同じライブラリ

## 7. スコープ外（Phase 2 以降）

以下の機能は Phase 1 では実装しない。将来的な拡張候補として記録。

### 7.1 機能拡張

- **ヘッダー/フッター付与**: ページ番号（「1/5」）や元URL自動印字
- **OCR連携**: 文字の途中で切れるのを100%防ぐ（Tesseract / EasyOCR 等）
- **分割数指定モード**: N等分で分割
- **幅方向の分割**: 横長画像への対応
- **複数画像の一括分割**: バッチ処理
- **出力形式変換**: PNG → JPEG 等（Image Converterとの統合）

### 7.2 高度なスマート分割

- **見出しタグ認識**: HTMLをスクレイピングしてキャプチャ時に見出し位置を記録
- **セクション自動認識**: 同じ背景色の塊を検出してセクション境界で分割
- **ヘッダー/フッター自動除外**: サイトヘッダー・フッターを認識して除外

### 7.3 UI拡張

- **ドラッグで分割位置を手動調整**
- **分割位置のマニュアル追加/削除**
- **各ピースの個別プレビュー**

## 9. 成功基準

以下の条件を全て満たせば Phase 1 完了とみなす:

- [ ] 縦長キャプチャ画像（5000px × 20000px 程度）を快適に分割できる
- [ ] スマート分割で文字の途中で切れるケースが大幅に減る
- [ ] 出力ファイル名が直感的（元ファイル名-part01 等）
- [ ] ZIPダウンロード正常動作
- [ ] モバイルでもレイアウト崩れなし
- [ ] Image Converter と同じ UI 設計言語で統一感あり
- [ ] プライバシー方針（サーバー送信ゼロ）が守られている

## 10. 気づきメモ・将来の課題

### 10.1 実装中に気づく可能性があるポイント

- Canvas の上限サイズ制限（ブラウザによって異なる、一般的に16384×16384）
- メモリ使用量（超縦長画像では制約あり）
- スマート分割の許容色差（固定値±10が妥当か、実画像で検証必要）
- 探索範囲（±50pxが適切か、画像の解像度に応じて調整検討）

### 10.2 マーケティング考慮点

- 「画像分割」「縦長 印刷」「スクリーンショット 分割」等のキーワード
- NoviqLab ブランドの2つ目のツールとして、ポートフォリオ価値を強化
