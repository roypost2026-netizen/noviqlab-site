## 3. Resend API を使ったお問い合わせフォームの実装

### フロントエンド（`app/contact/page.tsx`）

- `"use client"` ディレクティブを使用（フォームの状態管理のため）
- フォーム項目: 名前・メールアドレス・件名・本文
- `fetch` で `/api/contact` に POST リクエスト
- 送信状態を `idle / submitting / success / error` の4ステートで管理
- 送信成功後はページ遷移せずサンクスメッセージを表示、フォームをリセット

### バックエンド（`app/api/contact/route.ts`）

```ts
export async function POST(request: Request) {
  // ...バリデーション・Honeypot チェック...
  const resend = new Resend(apiKey);  // ← ハンドラ内でインスタンス化（重要）
  await resend.emails.send({ ... });
}
```

**注意点**: `Resend` インスタンスをモジュールトップレベルで生成すると、
ビルド時に `RESEND_API_KEY` が空の場合にエラーになる（`next build` が失敗する）。
ハンドラ関数の内部でインスタンス化することで回避。

### メール送信設定

```ts
resend.emails.send({
  from: "NoviqLab Contact <noreply@noviqlab.com>",
  to: [contactEmail],   // CONTACT_EMAIL 環境変数
  replyTo: email,       // 送信者のメールアドレスを Reply-To に設定
  subject: `[NoviqLab お問い合わせ] ${subject}`,
  text: `お名前: ${name}\nメールアドレス: ${email}\n\n${message}`,
});
```

`replyTo` に送信者アドレスを設定しているため、受信後そのまま返信可能。

## 4. Honeypot によるスパム対策

フォームに非表示フィールド `name="website"` を追加。

```tsx
<div className="hidden" aria-hidden="true">
  <input type="text" name="website" tabIndex={-1} autoComplete="off" />
</div>
```

- CSS で非表示にしているため通常ユーザーには見えない
- ボットが自動入力した場合、クライアント側・サーバー側の両方で検出・破棄
- サーバー側では `if (website) return Response.json({ ok: true })` として無害に処理

## 10. Vercel にカスタムドメインを DNS 設定

Vercel プロジェクト設定 → "Domains" → `www.noviqlab.com` と `noviqlab.com` を追加。

### ムームードメインで設定する DNS レコード

| タイプ | ホスト名 | 値 | 用途 |
|--------|----------|----|------|
| A | `@`（ルート） | `76.76.21.21` | noviqlab.com → Vercel |
| CNAME | `www` | `cname.vercel-dns.com` | www.noviqlab.com → Vercel |

**注意点**:
- DNS 反映には最大 48 時間かかる場合がある（通常は数分〜数時間）
- Vercel が SSL 証明書を自動発行・更新する（Let's Encrypt）
- ルートドメイン（`noviqlab.com`）は A レコード必須（CNAME 不可）

---

## 11. Resend に独自ドメイン登録・DNS 認証

Resend ダッシュボード → "Domains" → `noviqlab.com` を追加。

表示された DNS レコードをムームードメインに追加:

| タイプ | ホスト名 | 値 | 用途 |
|--------|----------|----|------|
| TXT | `resend._domainkey` | `p=...` | DKIM 署名（なりすまし防止） |
| TXT | `@` | `v=spf1 include:amazonses.com ~all` | SPF（送信元認証） |
| MX | `send` | `feedback-smtp.us-east-1.amazonses.com` | バウンス処理 |

DNS 反映後、Resend ダッシュボードで "Verify" をクリックして認証完了。

**注意点**:
- DKIM・SPF が未設定だとメールがスパム判定される可能性が高い
- ムームードメインで既に SPF レコードがある場合は既存レコードに `include:amazonses.com` を追記する
